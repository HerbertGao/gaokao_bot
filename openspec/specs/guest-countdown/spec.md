## 需求
### 需求:Guest 模式应答倒计时查询

当 Bot 在它并非成员的聊天中被 @提及或被回复而收到 Guest 消息（`Update.GuestMessage`）时，系统必须以一条默认模板的高考倒计时文本进行应答。系统必须使用该 Guest 消息携带的 `GuestQueryID`，通过 `answerGuestQuery` 方法发送应答。系统禁止为 Guest 交互写入任何持久化数据。

#### 场景:被 @提及且无参数

- **当** Bot 在非成员聊天中收到仅含 `@botname` 提及、无其他参数的 Guest 消息
- **那么** 系统必须查询当前北京时间所处时间范围内的所有考试
- **并且** 套用默认模板，把各考试倒计时拼接为单条文本
- **并且** 通过 `answerGuestQuery` 以该文本应答

#### 场景:被回复触发

- **当** Bot 在非成员聊天中因用户回复其消息而收到 Guest 消息
- **那么** 系统必须按无参数处理，应答当前时间范围内的默认倒计时

### 需求:Guest 模式支持指定年份参数

系统必须从 Guest 消息文本中剥离开头的 `@botname` 提及，得到可选的参数文本。当剩余参数为一个合法考试年份（在 `MinExamYear` 与 `MaxExamYear` 范围内）时，系统必须按该年份查询考试；当剩余参数为空时，系统必须按无参数处理。

#### 场景:指定合法年份

- **当** Bot 收到形如 `@botname 2026` 的 Guest 消息，且 2026 在支持年份范围内
- **那么** 系统必须查询 2026 年的考试并应答对应倒计时文本

#### 场景:参数非法或无法识别

- **当** Bot 收到的 Guest 消息参数不是合法年份（非数字、超出年份范围，或回复文本与查询无关）
- **那么** 当参数为空时系统必须回退为当前时间范围的默认倒计时
- **并且** 当参数非空但无法识别为合法年份时，系统必须应答「参数暂时无法识别。」

#### 场景:无可用考试数据

- **当** 查询结果中没有任何考试数据
- **那么** 系统必须应答提示文本，告知暂无可用信息

### 需求:Guest 模式仅提供默认倒计时

系统在 Guest 模式下禁止暴露 `/template`、`/debug` 命令及 Mini App 按钮，禁止使用用户自定义模板。Guest 应答必须仅使用默认模板，且必须以单条 `InlineQueryResult` 形式返回（`answerGuestQuery` 不支持多候选项）。

#### 场景:不暴露管理类功能

- **当** Bot 在 Guest 模式下应答任意查询
- **那么** 应答内容禁止包含 `/template`、`/debug` 的功能或 Mini App 跳转按钮
- **并且** 应答必须仅基于默认模板生成

#### 场景:应答失败的处理

- **当** `answerGuestQuery` 调用因超时或其他错误失败
- **那么** 系统必须记录错误日志且禁止重试，不得影响其他更新的处理
